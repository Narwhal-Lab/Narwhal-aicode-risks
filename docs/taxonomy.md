# Risk Taxonomy / 风险分类法

Cases in this repository are grouped into 7 categories. Each case's `meta.yaml` declares one primary `category`.

本仓库的案例按 7 大类组织。每个案例的 `meta.yaml` 中声明一个主类别 `category`。

| Key | Display | Description (EN) | 中文释义 |
|---|---|---|---|
| `supply-chain` | Hallucination & Supply Chain | AI hallucinates non-existent dependencies, recommends squatted/poisoned packages, or contaminates install-time supply chains. | AI 推荐不存在的依赖、被恶意抢注的包,或污染安装链路。 |
| `code-vulns` | Code-Level Vulnerabilities | Snippet-level defects in AI-generated code: unsafe API use, missing input validation, reintroduction of public CVE patterns. | AI 生成代码片段层面的缺陷:不安全 API 调用、输入校验缺失、公开 CVE 模式重引入。 |
| `cloud-iac` | Cloud & IaC Misconfiguration | Insecure-by-default infrastructure templates (Terraform, CloudFormation, etc.) generated by AI. | AI 生成的基础设施模板默认配置不安全(Terraform / CloudFormation 等)。 |
| `agent-risk` | Agent Risks | Prompt injection, tool-call hijacking, architectural biases that surface as security flaws in agent systems. | 提示注入、工具调用劫持、智能体架构层面的安全缺陷。 |
| `domain-specific` | Domain-Specific Risks | Risks unique to specific domains: smart contracts, AI app platforms, no-code generators, etc. | 特定领域(智能合约、AI 应用生成平台、no-code)的特有风险。 |
| `ip-compliance` | IP, License & Compliance | Copyright lawsuits, license contamination, training-data IP disputes around AI-generated code. | 版权诉讼、许可证污染、训练数据 IP 争议。 |
| `human-factor` | Human Factors | Skill erosion, over-reliance on AI, security culture decay in AI-assisted teams. | 开发者能力退化、对 AI 过度依赖、AI 协作团队安全文化退化。 |

## Severity scale / 严重度

We use a 5-point scale aligned with OWASP / GitHub Advisory conventions. Cases with assigned CVEs additionally record the CVSS v3.x score in `meta.yaml`. Color dots appear next to the severity label to give a quick visual cue.

采用 OWASP / GitHub Advisory 通用的五档评级。已分配 CVE 的案例额外在 `meta.yaml` 中记录 CVSS v3.x 评分。

- 🔴 `critical` — Trivial-to-exploit, large blast radius, no auth required.
- 🟠 `high` — Significant impact, narrow but real exploit conditions.
- 🟡 `medium` — Meaningful risk, requires specific preconditions.
- 🟢 `low` — Limited impact or strong mitigations available.
- ⚪ `info` — Awareness item; not directly exploitable but instructive.
