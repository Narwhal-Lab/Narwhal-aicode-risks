# 🛡️ Research on In-the-Wild Security Risks of AI-Generated Code

[English](README.md) | [中文](README_CN.md)

This repository contains the full text of the team’s technical report Research on In-the-Wild Security Risks of AI-Generated Code.

Based on real open-source project data and publicly disclosed vulnerability datasets, this study depicts the practical usage patterns and security risks of AI-generated code in real-world scenarios from an engineering practice perspective.

## 📚 Overview

With the popularization of large language models (LLMs) and generative AI, AI-powered programming tools have been deeply integrated into modern software engineering workflows—evolving from intelligent code completion and boilerplate code generation to the implementation of module-level code and complex logic. AI is transforming from a "supporting tool" into a new code contributor, which has also significantly reshaped the security boundaries of the software supply chain.

This study is based on two types of real-world datasets:

- **Historical Records of Open-Source Code Repositories**: Analyze the commit histories of mainstream open-source projects to observe the usage trends and evolutionary characteristics of AI-generated code across different programming languages and development phases.
- **Disclosed Vulnerabilities and Remediation Commits**: Focus on CVE-related remediation activities to characterize the actual role and impact of AI-generated code throughout the vulnerability lifecycle, from introduction to remediation.

## ✨ Key Findings

- **Phased Penetration**: 

AI-generated code has undergone three distinct phases of penetration in the open-source ecosystem: Explosive Exploration → Rational Regression → Stable Collaboration. Ultimately, it is more concentrated in high-repeatability, easily verifiable tasks such as test code writing, documentation drafting, and boilerplate code generation.

- **Significant Disparities Across Language Stacks**: 

Developers exhibit notable caution when using AI-generated code in system-level languages like Rust and C++, in contrast to its high penetration in languages with large ecosystems and flexible syntax, including Python, JavaScript, and TypeScript.

- **Dual Role in the Vulnerability Lifecycle**: 

In certain security-sensitive scenarios, code originally generated by AI is replaced with manual implementations during remediation. Meanwhile, a considerable proportion of vulnerability fixes are completed using AI-generated code. Thus, AI can act as both a source of vulnerabilities and an accelerator for remediation.

- **Patterned Risk Characteristics**: 

Vulnerabilities introduced by AI are concentrated in snippet-level defects, such as inadequate input validation, unsafe API calls, and outdated cryptographic implementations. The severity distribution of these vulnerabilities is highly similar to that of manually written code, but the risks are more prominent in network-exposed logic related to APIs, web services, and network protocols.

- **Proposed Three-in-One Mitigation Framework**: 

A defense-in-depth approach is constructed from three dimensions: establishing multi-dimensional security evaluation benchmarks, enhancing the intrinsic security of AI models, and implementing human-machine collaborative governance. The framework emphasizes that developers bear ultimate security responsibility for AI-generated code.
